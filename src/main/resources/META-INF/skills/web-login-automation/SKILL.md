# 通用网页登录自动化技能

## 概述

此技能用于自动化任意网站的登录流程。支持参数化配置URL、用户名、密码，智能识别登录表单元素，处理各类验证码，登录后执行用户指定的操作。

## 参数说明

| 参数 | 必需 | 说明 |
|------|------|------|
| url | 是 | 登录页面地址 |
| username | 是 | 登录用户名 |
| password | 是 | 登录密码 |
| action | 否 | 登录后要执行的操作描述 |

## 工作流程

### 1. 启动浏览器并访问登录页面

```
1. 调用 browser_start 启动浏览器
2. 调用 browser_action 导航到用户指定的 URL
3. 调用 browser_view 分析页面结构
```

### 2. 智能识别登录表单

自动识别以下元素：
- 用户名输入框：input[type="text"], input[name*="user"], input[name*="account"], input[placeholder*="用户"], input[placeholder*="账号"]
- 密码输入框：input[type="password"]
- 登录按钮：button[type="submit"], button:has-text("登录"), button:has-text("Login"), input[type="submit"]

### 3. 填写登录信息

```
操作步骤：
1. 在用户名输入框填写用户提供的用户名
2. 在密码输入框填写用户提供的密码
3. 点击登录按钮
```

### 4. 处理验证码

**验证码类型及处理方式：**

| 验证码类型 | 处理方式 |
|-----------|---------|
| 图片选择验证码 | 提示用户在浏览器中手动完成选择 |
| 滑块验证码 | 提示用户手动拖动滑块 |
| 文字识别验证码 | 提示用户输入验证码 |
| 短信验证码 | 提示用户输入收到的验证码 |

处理流程：
1. 调用 browser_view 检测是否有验证码
2. 告知用户验证码类型和提示内容
3. 请用户在浏览器中手动完成验证码
4. 用户确认完成后继续后续操作

### 5. 验证登录状态

```
登录成功判断：
1. URL 发生变化（跳转到主页或仪表盘）
2. 页面出现用户信息或退出按钮
3. 没有错误提示信息

登录失败处理：
1. 检查错误提示信息
2. 告知用户失败原因
3. 询问是否重试
```

### 6. 执行登录后操作

根据用户指定的 action 参数执行相应操作：
- 查询数据：导航到相应页面，提取数据
- 点击操作：找到目标元素并点击
- 表单填写：定位表单并填写信息

## 使用示例

**示例1：登录商城后台查询商品**
```
用户：帮我登录 https://multishop-test.dlchm.cn/login，用户名 fccc1，密码 123456a@A，登录后查看商品列表前5条

执行步骤：
1. 启动浏览器，访问登录页面
2. 填写用户名 fccc1 和密码 123456a@A
3. 提示用户完成验证码（如有）
4. 用户确认后，导航到商品列表页面
5. 提取前5条商品名称
```

**示例2：登录管理后台**
```
用户：登录 https://admin.example.com，账号 admin，密码 admin123

执行步骤：
1. 启动浏览器，访问登录页面
2. 填写账号密码
3. 点击登录
4. 确认登录成功
```

**示例3：登录后执行操作**
```
用户：登录 https://crm.example.com，用户名 sales，密码 pass123，登录后查看今日订单

执行步骤：
1. 登录系统
2. 导航到订单页面
3. 筛选今日订单
4. 提取订单信息
```

## 页面元素识别策略

### 用户名输入框选择器优先级
1. input[name="username"]
2. input[name="user"]
3. input[name="account"]
4. input[type="text"]:first
5. input[placeholder*="用户"]
6. input[placeholder*="账号"]
7. input[placeholder*="手机"]

### 密码输入框选择器优先级
1. input[name="password"]
2. input[type="password"]

### 登录按钮选择器优先级
1. button[type="submit"]
2. button:has-text("登录")
3. button:has-text("Login")
4. input[type="submit"]
5. .login-btn
6. #login-btn

## 错误处理

| 错误类型 | 处理方式 |
|---------|---------|
| 找不到用户名输入框 | 尝试备选选择器，或提示用户 |
| 找不到密码输入框 | 尝试备选选择器，或提示用户 |
| 找不到登录按钮 | 尝试备选选择器，或提示用户 |
| 验证码失败 | 提示用户重新完成验证码 |
| 登录超时 | 刷新页面重新尝试 |
| 账号密码错误 | 告知用户，询问正确凭据 |
| 网络错误 | 检查网络连接后重试 |

## 注意事项

1. **验证码处理**：所有验证码必须由用户手动完成，AI只能提示用户操作
2. **登录状态**：浏览器会保持登录状态，后续操作无需重复登录
3. **安全性**：不要在日志中输出用户密码
4. **兼容性**：不同网站的登录表单结构可能不同，需要灵活识别

## 输出格式

登录成功后，根据用户请求输出相应结果：

```
✅ 登录成功！

网站：[网站名称/URL]
账户：[用户名]

[执行的操作结果]
```
